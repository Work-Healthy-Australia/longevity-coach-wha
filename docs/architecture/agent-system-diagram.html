4
Weekly Research Digest → Vector DB → Janet's RAG Layer
The health_researcher pipeline is the sole writer to the knowledge base. Janet hybrid-searches it at every session start, grounding responses in current evidence.
PHASE 1
Web Search
6 parallel web_search calls — PubMed, Nature, Cell, medRxiv, Cochrane. Top 5 URLs + snippets per category.
~10 s
PHASE 2
Fetch + Extract
Up to 30 article bodies fetched in batches of 6. Deduplicates against existing health_knowledge (last 30 days).
~25 s
PHASE 3
Synthesize
3 parallel LLM calls × 2 batches. One call per category → structured digest + 3–5 key passages for embedding.
~40 s
PHASE 4
Chunk + Embed
400-token chunks, 80-token overlap. Batch embed via perplexity/pplx-embed-v1-4b (2560 dims) through OpenRouter.
~20 s
PHASE 5
Upsert + Prune
Insert to health_updates. Upsert chunks to health_knowledge. Prune rows older than 90 days.
~5 s
🔍
Next Janet session — how the knowledge reaches the patient
PatientContext.load() assembles a profile query string from the user's top 2 risk domains + last check-in topic (no LLM call, ~0 ms). OpenRouter embeds it (~20 ms). hybrid_search_health() runs HNSW vector + BM25 full-text, fuses ranks with RRF (~10 ms). Top-5 chunks are injected into Janet's system prompt as a RELEVANT RESEARCH block. Total overhead: ~30 ms — runs in parallel with all other PatientContext reads, adding zero sequential latency.
Janet cites evidence from the knowledge base with appropriate confidence based on each chunk's evidence_level metadata (strong / moderate / preliminary).
